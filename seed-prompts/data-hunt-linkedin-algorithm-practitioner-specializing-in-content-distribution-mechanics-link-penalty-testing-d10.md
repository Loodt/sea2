## Key Decisions
LinkedIn does NOT differentiate external link penalties by destination domain. The 40-60% reach penalty is binary — triggered by presence of any external URL in the post body, regardless of whether it points to GitHub, documentation, or a commercial landing page. This conclusion is supported by three converging evidence lines: (1) the GR ranking model's feature set contains no URL-destination or domain-level signals, (2) Hootsuite's 184-post experiment and multiple practitioner analyses (AuthoredUp, van der Blom, Gromming, Botdog) all document uniform penalties without domain segmentation, and (3) the penalty mechanism operates at the crawler/indexing level which scans for URL presence, not URL quality. For developer-tool marketing, this means GitHub repo links, API docs, and commercial landing pages all receive identical treatment — making zero-click native content the universal optimal strategy, with links relegated to first comments (5-15% penalty) or profile featured sections (0% penalty).
